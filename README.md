A complex, Agent-Based Modeling (ABM) simulation built with Python and Mesa. This project simulates the unpredictable spread of forest fires and the autonomous, AI-driven response of firefighting units. The environment and agents are visualized in real-time through an interactive Solara web dashboard.

## ✨ Key Features

### 🤖 AI-Driven Agents
* **Fire Engines (State Machines):** Units autonomously transition between states (Dispatch, Extinguish, Return to Base, Refill). They utilize grid-based pathfinding to navigate, prioritize high-intensity fires, and strategically create "Firebreaks" (wet zones) to halt the fire's advance.
* **Scouter Drones:** Unmanned aerial vehicles patrol the map to detect new ignitions. Their visibility range is dynamically impaired by the thickness of the smoke generated by active fires.

### 🌲 Dynamic Environment & Physics
* **Complex Forest Ecosystem:** Trees are not uniform blocks. They possess unique attributes such as moisture levels, species (e.g., Pine burns faster, Oak is resistant), and dynamic heat resistance.
* **Vector-Based Fire Spread:** The probability of fire spreading to adjacent cells is calculated using real-time vectors, including:
  * **Wind:** Direction and strength (capable of creating "Spot Fires" where embers jump across the map).
  * **Topography:** Fire travels 1.5x faster uphill and slower downhill.
  * **Weather & Time:** Temperature, humidity, and Day/Night cycles dynamically affect ignition probabilities.

### ♾️ Perpetual Simulation Engine
* Once all fires are contained, the simulation enters a "cooldown" phase before a lightning strike or new ember randomly sparks a new fire, ensuring continuous, endless operation without manual resets.

---

## ⚙️ How It Works (Under the Hood)

The simulation relies on a combination of **Cellular Automata** (for environmental spread) and **Agent-Based Modeling** (for the physical entities). 
* **Firebreaks:** When a fire unit extinguishes a burning tree, it drastically reduces the `moisture` level of surrounding green trees, effectively creating a firebreak. The spread algorithm detects this and prevents embers from landing in controlled zones.
* **Dispatch System:** A centralized function evaluates all known fires, assigns a priority score based on heat intensity and distance, and dispatches the closest available idle fire unit.

## 🚀 Installation & Usage

It is highly recommended to run this project inside a Python Virtual Environment. The code is optimized for **Python 3.14** but is backwards compatible.

**1. Clone the repository**
```bash
git clone [https://github.com/Alex247Git/wildfire-simulation.git](https://github.com/Alex247Git/wildfire-simulation.git)
cd wildfire-simulation
```

**2. Create and activate a Virtual Environment**
* On Windows:
  ```bash
  python -m venv venv
  venv\Scripts\activate
  ```
* On macOS/Linux:
  ```bash
  python3 -m venv venv
  source venv/bin/activate
  ```

**3. Install dependencies**
```bash
pip install -r requirements.txt
```

**4. Launch the Simulation**
```bash
py -3.14 -m solara run app.py
```
*(The UI will automatically open in your default web browser, usually at `http://localhost:8765`)*
